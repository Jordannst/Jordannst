# GitHub Profile README Redesign

## Goal

Redesign Jordan's GitHub profile README into a cleaner, more professional profile that fits an IT student building toward software engineering opportunities.

The new README should feel focused, mature, and easy to scan. It should avoid the overly decorative style common in junior profile READMEs while still showing personality through clear positioning and thoughtful wording.

## Audience

Primary audience: recruiters and engineers who quickly scan GitHub profiles.

The README should help them understand:

- who Jordan is
- what Jordan is focused on
- what technologies Jordan currently works with
- where to find portfolio/contact links

## Positioning

Jordan should be presented as:

> An IT student focused on building mobile-first apps, modern web experiences, and cloud-backed products.

Tone should be:

- professional
- concise
- humble
- builder-oriented
- not exaggerated into senior-level branding

## Content Strategy

Use a text-first layout. Replace decorative visual elements with concise sections.

Remove or avoid:

- animated header/banner
- waving emoji heading
- typing SVG animation
- profile view counter
- long technology icon wall
- decorative GIF
- snake animation
- non-clickable social badges
- excessive badges or visual noise

Keep or add:

- name/title intro
- short positioning paragraph
- portfolio, LinkedIn, and email links
- focus areas
- curated tech stack
- selected work placeholder

```md
# Jordan

IT student focused on building mobile-first apps, modern web experiences, and cloud-backed products.

Currently working with React Native, React, Astro, Node.js, Supabase, and PostgreSQL.
```

## Tech Stack Guidance

The README should still include a stack, but it must be curated. The previous long icon list makes the profile feel noisy and less focused.

Recommended stack display:

```md
React Native · React · Astro · TypeScript · Node.js · Supabase · PostgreSQL · Go
```

This gives enough signal without turning the README into a technology dump.

## Links

Use simple text links instead of large badge rows:

- Portfolio: `https://www.jordannst.dev/`
- LinkedIn: `https://www.linkedin.com/in/jordannst08/`
- Email: `mailto:jordannst.08@gmail.com`

Instagram, Discord, and Facebook should be omitted unless there is a strong reason to include them professionally.

## Selected Work

If there are no confirmed projects to highlight yet, include a lightweight placeholder:

```md
## Selected Work

Coming soon.
```

This keeps the structure ready for future improvements without inventing project claims.

## Success Criteria

The redesigned README succeeds if it:

- feels professional and clean at first glance
- can be scanned in under 10 seconds
- clearly communicates Jordan's current focus
- avoids excessive animation and decoration
- presents a realistic IT-student profile
- leaves room to add selected projects later

## Non-Goals

This redesign will not:

- create fake project claims
- overstate seniority
- add complex generated graphics
- optimize for maximum badges/stats
- include every technology Jordan has ever used
